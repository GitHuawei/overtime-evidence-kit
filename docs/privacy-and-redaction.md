# 隐私与脱敏原则

## 永不进入仓库的内容

- 真实聊天记录。
- 真实 Git 提交。
- 真实录音或录音转写。
- 真实公司、人名、项目、客户、地址、手机号。
- 身份证号、银行卡号、工资流水原文。
- 合同扫描件、源码、客户数据、商业秘密。
- 可反推出真实案件事实的日期、金额或组合细节。

## 脱敏策略

- 人员：使用 `员工-A`、`主管-B`、`HR-C`。
- 公司：使用 `示例科技有限公司`、`A 公司`。
- 项目：使用 `订单系统`、`BI 报表服务`、`客服后台`。
- commit：使用 `mock-a1b2c3d` 格式。
- 金额：使用显然示例化的整数或 `0`。
- 地址：使用 `示例城市示例区域`。

## 公开输出原则

公开输出只保留证据链所需的最小信息。若某项信息不是证明时间、任务、结果或来源所必需，应删除或泛化。

## 暂停条件

遇到以下情况应暂停：

- 用户要求伪造、篡改、补写记录。
- 用户不能说明材料来源。
- 用户要求保留无必要隐私或商业秘密。
- 用户要求承诺结果。
- 原始材料缺失到无法形成可追溯证据链。
