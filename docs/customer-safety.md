# Customer Safety

如果你对自己的材料有疑问，请先保护隐私，再讨论需求。

## 不要公开上传

不要在 GitHub issue、PR、discussion、commit、附件或截图中上传或粘贴：

- 真实微信聊天、群聊、截图、导出记录或 message ID。
- 真实 Git commit、仓库名、分支名、源码或 commit hash。
- 真实录音、转写、会议纪要或文件名。
- 真实公司名、人名、项目名、客户名、地址、手机号、合同、工资、金额或身份标识。
- 任何能够识别真实案件、真实工作场所、真实客户、真实项目或真实个人的信息组合。

## 如果想咨询

先只描述你希望解决的问题类型，不要发送原始证据。等待私下渠道、授权范围、材料最小化、存储方式和删除规则都确认后，再决定是否继续。

## 安全提问方式

可以问：

- 这个项目如何组织 mock evidence package？
- 如果我有多类材料，应该先梳理哪些结构问题？
- 如何在不发送原始材料的前提下说明我的需求？

不要问：

- 直接帮我判断能不能赢。
- 我把聊天记录贴到 issue 里你帮我看。
- 我上传真实录音你帮我转写。

## 边界

本项目不提供法律意见，不承诺结果，不在公开仓库处理真实证据。
