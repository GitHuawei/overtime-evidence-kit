# FAQ

## 这个项目能处理我的真实微信聊天记录吗？

不能。`overtime-evidence-kit` 是 mock-only 开源项目，公开仓库不接收、不处理、不存储真实微信、真实 Git、真实录音或任何可识别真实案件的信息。

## 为什么坚持 mock-only？

加班证据通常包含个人隐私、公司信息、项目细节、时间线、金额、合同和沟通内容。把真实材料放进公开仓库、issue、PR 或 discussion 会带来隐私和合规风险。mock-only 能让大家讨论结构、校验和输出，而不暴露真实材料。

## 为什么不直接把证据丢给 AI 总结？

证据整理的关键不是堆材料，而是把任务来源、处理过程、结果反馈、时间窗口和证据定位组织成可追踪、可复核的结构。这个项目演示的是结构化方法，而不是替代专业判断。

## 这个项目是不是法律意见？

不是。本项目不提供法律意见，不生成法律结论，不承诺协商、仲裁、诉讼或执行结果。

## demo 输出能直接提交仲裁吗？

不能。demo 输出来自完全虚构的 mock 数据，只能用于结构演示和本地验证。真实事项需要由当事人和专业人士在安全、授权、合规的前提下复核。

## CSV 为什么可以用 Excel 打开？

`mock-evidence-index.csv` 使用 Excel-friendly UTF-8 with BOM。这样 Windows Excel 双击打开时更容易正确识别中文。

## JSON package 和 Markdown/CSV 有什么区别？

`package.json` 保留机器字段和机器枚举值，例如 `eventType=weekday_overtime`。Markdown report 和 CSV evidence index 面向人工阅读，会显示中文标签，例如 `工作日加班`、`需复核`、`群聊`。

## 如果我想咨询自己的材料，怎样才安全？

不要在 GitHub issue、PR、discussion 或公开附件中粘贴真实材料。先询问是否有私下联系渠道，并先确认授权、隐私边界、材料范围、保留/删除规则和是否需要律师参与。任何咨询都不应被理解为法律意见。
